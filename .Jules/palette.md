## 2026-04-24 - [Native Tooltips and Focus States]\n**Learning:** Sighted users can struggle with icon-only buttons just as much as screen reader users if tooltips aren't provided. Relying solely on `aria-label` helps a11y but ignores standard mouse hover UX. Furthermore, when building custom toggle buttons (like mobile menus), dynamically updating `aria-label` and using `aria-expanded` is crucial for screen readers, as the visual change (hamburger to X) isn't inherently semantic. Also, using `focus-visible:ring-2` instead of global `focus:ring-2` provides essential keyboard navigation visibility without unnecessarily altering the standard mouse clicking experience.\n**Action:** For all icon-only buttons, consistently pair `aria-label` with the native `title` attribute so both accessibility and mouse-hover use cases are satisfied. Use `focus-visible` utility classes for interaction-safe focus rings, and ensure stateful buttons update their `aria-` properties dynamically based on state.
